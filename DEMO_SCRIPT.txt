[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture of the system.
Our central orchestrator is the board_game_hub_agent, which manages all board game logic.
While this version runs a single main agent, the architecture is designed to support specialized sub-agents for distinct tasks like rules parsing or scheduling.
Before any query reaches the agent, it must pass through the security_checkpoint, which filters and sanitizes inputs.
The agent then calls our custom MCP tools, like search_board_game and organize_gaming_session, to interact with the Kaggle dataset and local JSON databases.
Finally, human approval is built into the workflow, allowing the user to resolve name ambiguities conversationally.

Finally, let's verify our video game filter.
